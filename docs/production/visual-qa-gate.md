# 视觉 QA Gate

最后更新：2026-06-25

本文件把“不要再让用户指出明显 UI 瑕疵”变成固定检查项。任何玩家可见改动，尤其是手机竖屏 UI、对话、战斗、背包、装备、主菜单、结算界面，都要过这一关。

## 必须检查

### 布局

- 主要内容不被 HUD、按钮、系统栏遮挡。
- 手机竖屏以 720x1280 为基准验证。
- 地图视窗是局部裁切的大地图，不是把整张地图缩小塞进屏幕。
- 浮层只占必要空间，不压住关键视觉焦点。
- 同组按钮高度、宽度、内边距一致。

### 文本

- 中文自然，不使用“新手入口村”这类生硬占位名。
- 学习目标词可用强调色，但强调色不能破坏可读性。
- 文本不能超出容器，不能贴边，不能被按钮遮挡。
- 英文只出现在单词学习选项、词条、专有名词中。
- 手机端不出现“空格/E”等键盘操作提示。

### 素材

- 容器、边框、按钮、名牌来自素材库或生成素材。
- 不使用临时代码绘制边框冒充最终 UI。
- 素材是完整物体，不截半棵树、半个房子、半个道具。
- 新素材必须记录来源、许可和用途。

### 交互

- 鼠标/触摸点击按钮应触发对应动作，不被地图寻路抢走。
- 对话、挑战、拾取、进入下一层都应有触摸友好的可点击目标。
- 交互提示只在可交互范围内出现。
- 已完成或已拾取对象不应留下误导性虚影。

## 截图验收

每次视觉改动至少生成一个相关截图。截图必须人工检查以下问题：

- 文字是否完全在容器内。
- 按钮是否压住文字或超出面板。
- HUD 和小地图是否压住可玩区域。
- 角色和怪物是否在合理视觉焦点内。
- 所有容器是否来自素材，而不是简单色块边框。

当前可用命令：

```bash
scripts/run_visual_gate.sh pc_village
scripts/run_visual_gate.sh word_castle_portrait
```

输出截图：

```text
.tmp_assets/main_menu_visual_pass.png
.tmp_assets/village_visual_pass.png
.tmp_assets/word_castle_portrait_prototype.png
```

## 失败处理

只要截图里能肉眼看到问题，就不要解释为“代码上没问题”。直接修布局、素材或文案，然后重新跑截图。
