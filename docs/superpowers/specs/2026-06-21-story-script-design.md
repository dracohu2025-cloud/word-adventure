# 故事剧本设计

日期：2026-06-21

状态：方向已确认，可进入章节剧本拆分

## 目标

为 `Word Adventures` 定义长期故事框架，让词汇学习、RPG 成长、NPC 任务、战斗、装备奖励和未来语言包共享同一套叙事逻辑。

故事框架需要支持：

- 温暖童话冒险的情绪基调，以及王道 RPG 的成长动力。
- 中国大陆小学和初中学生作为第一目标用户。
- 英语作为第一套学习语言卷。
- 长期扩展到 1500-3000 个词。
- 后续替换或新增任意词库，包括日语、德语或自定义词库。
- 视觉优先生产：优先使用 Tiny Swords，其次使用许可安全的开源素材，最后再考虑生成素材。

## 已确认的创意方向

### 基调

游戏采用：

- 温暖童话冒险作为情绪底色。
- 王道 RPG 成长作为玩家动力。

世界应该足够安全、明亮，适合低龄学习者；同时也要有村庄、森林、怪物、装备、战斗和探索，支撑真正的冒险感。

### 主角

主角是一个普通孩子，被魔法书选中成为见习勇者。

这个设定提供两层代入感：

- “我只是普通学习者，从小处开始也没关系。”
- “我可以通过学习、帮助别人和不断练习，成长为真正的勇者。”

第一版实现只做战士职业，但故事不应该暗示战士是未来唯一职业。

### 语言世界观

世界由不同语言中的“真名”维持。

同一事物可以拥有多种语言真名：

- 中文：`书`
- 英语：`book`
- 日语：`本`
- 德语：`Buch`

当前第一卷聚焦英语真名，但世界本身不是英语唯一。

### 词灵

单词也是记录在《万语之书》里的小精灵。

当书失控后：

- 一些词灵变成散落的书页。
- 一些词灵藏进破碎的句子。
- 一些词灵变成小怪物。
- 更强的词灵变成守关 Boss。

学习一个词，不被叙述成孤立背诵，而是理解一个事物的真名，并帮助迷路的词灵回到书中。

## 核心故事前提

很久以前，《万语之书》记录着世界万物在不同语言中的真名，并维持这些真名的平衡。

某一天，词灵王失踪了。失去词灵王后，《万语之书》的各个语言卷开始散落。村庄忘记了日常词语，森林忘记了动物和植物的名字，工具失去用途，小词灵们也变成了迷茫的怪物。

主角被《万语之书》选中，成为见习勇者。主角需要走过一个个地区，修复散落的语言卷，帮助词灵记起自己是谁，并最终找出词灵王失踪的真相。

## 长期主线

主线由两条弧线组成：

1. 修复《万语之书》。
2. 寻找失踪的词灵王。

这样设计可以同时提供结构和悬念：

- 修复书卷提供清晰的学习进度。
- 寻找词灵王提供长期故事动机。

## 内容容量模型

故事必须能容纳 1500-3000 个词，同时不能让每个词都挤进主线对话。

### 词汇层级

| 层级 | 功能 | 参考容量 |
| --- | --- | --- |
| 主线词 | 推动剧情和解锁进度的必学词 | 每个小章节 10-20 个 |
| 支线词 | NPC 支线、装备、材料、地区任务 | 每个地区 30-60 个 |
| 复习词 | 通过遭遇、宝箱、对话和书本复习反复出现 | 每个地区 40-100 个 |
| 收集词 | 可选词灵、图鉴、成就或未来复习系统 | 弹性扩展 |

### 地区容量

| 范围 | 目标词汇容量 |
| --- | --- |
| 新手村 | 12-20 个活跃词 |
| 小地区 | 80-150 个词 |
| 大章节 | 300-500 个词 |
| 第一部完整冒险 | 1500-3000 个词 |

关键规则：每个地区可以包含很多词，但只有少量词应该阻塞主线进度。

## 第一部地区规划

下面的地区规划用于承载 1500-3000 个词，同时尽量贴近奇幻 RPG 的视觉语言。

| 地区 | 剧情功能 | 词汇主题 | 素材可行性 |
| --- | --- | --- | --- |
| 书灯村 | 新手引导和前三页书页 | 家、朋友、书、水、剑、盾、简单动作 | 当前 Tiny Swords 素材支撑较强 |
| 低语森林 | 第一张野外地图 | 动物、植物、颜色、方位、简单形容词 | Tiny Swords 地形、树木、敌人、装饰支撑较强；动物素材可能需要补充 |
| 星灯湖畔 | 日常生活和恢复主题 | 食物、饮水、天气、身体、感受 | 中等；水域、食物、身体图标可能需要开源素材或生成补充 |
| 铁砧矿道 | 装备和材料成长 | 工具、材料、数字、职业、交易 | 中等；工具和矿洞视觉可能需要开源素材 |
| 风车农场 | 日常动作和学校化表达 | 时间、家务、学校物品、常用动词 | 中等；农场素材部分可覆盖，学校物品大概率需要补充 |
| 云上图书塔 | 复习与抽象词过渡 | 形容词、抽象名词、句型 | 中等；图书和 UI 可组合，塔内场景可能需要补充 |
| 旧王城 | 社会关系和规则 | 城市、规则、情绪、关系 | 中低；城市差异化和 NPC 多样性可能需要更多素材 |
| 遗忘边境 | 后期综合复习挑战 | 高频考试词、混合语法、Boss 复习 | 弹性较高；可复用奇幻战斗资产和新 UI 设计 |

这张表是生产约束。如果某个地区需要大量 Tiny Swords 或已批准开源素材无法支撑的视觉对象，就应该推迟、简化或重写该地区。

## 素材可支撑性策略

### 当前 Tiny Swords 覆盖

项目内已选择导入的 Tiny Swords 运行时素材覆盖：

- 地形。
- 村庄建筑。
- 战士主角动画。
- Pawn 和 Monk NPC 动画。
- Skull 敌人动画。
- 树木、石头、南瓜、稻草人、路牌、花盆、木栅栏。
- UI 面板、横幅、按钮、血条、图标、物品图标和头像。

本地 `external_assets/Tiny Swords` 原始素材目录还显示出更大的覆盖面：

- Free Pack。
- Enemy Pack。
- Update 010。
- Deco。
- Effects。
- Factions。
- Resources。
- Terrain。
- UI。

因此，Tiny Swords 足够支撑第一版新手村、早期森林、基础战斗、奇幻 UI 和许多第一部冒险道具。

### 当前开源素材覆盖

项目已经记录的可用兜底素材包括：

- Kenney Tiny Town。
- Kenney Tiny Dungeon。
- Kenney Game Icons。
- OpenGameArt 80 CC0 RPG SFX。
- MiniMax 生成的草稿 BGM。

Kenney 素材尤其适合做 CC0 兜底图标、简单道具、瓦片缺口补位和历史原型。但除非经过视觉评审，不应该把 Kenney 素材混入 Tiny Swords 的焦点构图中。

### 已知缺口

当前素材池可能无法完全覆盖：

- 现代学校物品。
- 丰富城市室内场景。
- 大量动物种类。
- 湖泊生态。
- 差异化矿洞室内。
- 多种独特怪物家族。
- 日语、德语或其他未来语言包所需的文化特定道具。
- 情绪、规则、思想、语法概念等抽象词的视觉表达。
- 大量独特装备图标。

这些缺口不阻止故事框架成立，但必须约束章节顺序和视觉实现范围。

### 生产规则：素材门槛

任何新地区进入实现前，都必须先通过素材门槛。

针对一个拟实现地区，先列出：

- 必需地形。
- 必需建筑。
- 必需 NPC 类型。
- 必需敌人类型。
- 必需道具。
- 必需 UI 状态。
- 必需 SFX / BGM。

然后把每一项分类为：

- Tiny Swords 已覆盖。
- 现有已批准开源素材已覆盖。
- 需要新引入许可安全的开源素材。
- 需要生成素材。
- 应该移除或重写。

目标是：主路径和关键焦点对象在实现前必须具备可用素材方案。

### 避免“一词一素材”爆炸

故事不应该要求每个词都拥有一个独立视觉资产。

更可行的映射方式：

- 一个地区通过共享环境教授一组相关词。
- 一个 NPC 教授一个词簇。
- 一个物品图标代表一类物品。
- 一个敌人家族代表多个迷失词灵。
- 《万语之书》可以通过 UI 展示抽象或难以具象化的词，不必放置独立地图对象。

这样才能让 1500-3000 个词的长期目标保持可行。

## 语言包模型

即使第一版内容是英语，故事也应该保持语言无关。

推荐概念模型：

```text
万语之书
-> 语言卷
-> 地区
-> 任务线
-> 词集
-> 词条
```

### 词条字段

未来词条数据应支持：

- `target_language`：English、Japanese、German、custom。
- `source_language`：第一目标用户默认 Chinese。
- `term`：`book`、`本`、`Buch` 等。
- `meaning`：中文含义。
- `pronunciation`：可选。
- `part_of_speech`：可选。
- `topic`：village、forest、food、school 等。
- `difficulty`：课标、CEFR、JLPT、自定义或项目内部难度。
- `examples`：可选例句。
- `asset_hint`：可选视觉分类。
- `challenge_types`：释义、拼写、语境、听力、复习。

剧本应该使用“真名”和“语言卷”这样的表达，而不是“英语是唯一魔法”。

## 第一章：书灯村

详细可制作剧本见：

- `docs/story/chapter-01-adventure-gate-village-script.md`

第一章和第二章地图设计、章节转换设计见：

- `docs/story/chapter-01-02-map-design.md`

### 目的

书灯村教授：

- 如何移动。
- 如何和 NPC 对话。
- 如何完成词汇挑战。
- 学习如何修复书页。
- 奖励如何通过 HUD、背包、装备和战斗可视化。
- Boss 遭遇如何打开下一个地区。

### 剧情节拍

1. 主角在村庄道路附近醒来，看到《万语之书》产生反应。
2. 书告诉主角：英语卷最先散落了。
3. 图书管理员请求主角找回 `book` 对应的书页。
4. 铁匠请求主角恢复 `sword` 这个真名。
5. 园丁或井边村民请求主角通过语境恢复 `water`、`shield` 或另一个日常词。
6. 三页书页修复后，森林门解除封印。
7. Word Imp 作为第一个迷失词灵出现。
8. 玩家通过 RPG 战斗击败 Word Imp。
9. Word Imp 回到《万语之书》，并暗示词灵王失踪。
10. 森林道路开启。

### 第一章词池

第一章词量要保持小而清晰。

核心词：

- `book`
- `sword`
- `shield`
- `water`
- `home`
- `friend`

可选支撑词：

- `tree`
- `stone`
- `fire`
- `light`
- `forest`
- `gate`
- `apple`
- `river`
- `sun`
- `eat`
- `drink`
- `help`

只有其中一部分应该作为必需词。

### NPC 角色

| NPC | 剧情功能 | 学习功能 | 奖励 |
| --- | --- | --- | --- |
| 图书管理员 | 解释《万语之书》 | 释义识别 | 书页 1 |
| 铁匠 | 把学习和装备力量绑定 | 拼写回忆 | 书页 2、武器或攻击奖励 |
| 园丁 / 井边村民 | 把词语和日常生活连接起来 | 语境填空 | 书页 3、药水或防御奖励 |
| Word Imp | 第一个迷失词灵 | 剧情高潮和战斗反馈 | 森林门开启、金币 / 经验 / 可能的物品 |

### 对话风格

对话应该：

- 中文优先。
- 对低龄学习者足够短。
- 温暖、具体、任务导向。
- 只有当词语本身是学习对象时才出现双语内容。

示例：

```text
图书管理员：这本书不是普通的书。
图书管理员：它叫《万语之书》，记录着世界上许多事物的真名。
图书管理员：今天我们先找回英语卷里的第一个真名：book。
```

## 剧本生产格式

每个章节剧本都应该拆成：

- 章节前提。
- 地区目标。
- 必需词集。
- NPC 列表。
- 任务节拍。
- 对话台词。
- 挑战题面。
- 奖励。
- 素材需求。
- 测试 / 截图需求。

这样可以让故事写作始终服务于生产，而不是停留在设定层。

## 非目标

本文档不设计完整 3000 词课程。

本文档不要求每个词都有独立素材。

本文档不把故事锁死为英语唯一魔法。

第一部冒险不引入多主角、复杂道德选择、恋爱线或黑暗主题。

第一章不加入暂时用不到的大系统，例如语音演出、分支结局或完整 lore 档案馆。

## 验收标准

故事框架满足以下条件时，可以进入章节剧本拆分：

- 第一章可以基于本文档写出，不需要重新发明故事前提。
- 长期结构可以合理容纳 1500-3000 个词。
- 英语是第一语言卷，但不是唯一可用语言。
- 新地区加入前必须考虑素材可行性。
- Tiny Swords 继续作为早期章节主美术家族。
- 开源素材和生成素材有清晰的兜底角色。
- NPC 任务、词汇挑战、奖励和战斗都服务于同一个故事前提。
