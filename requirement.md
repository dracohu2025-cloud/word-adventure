# Word Adventures 需求基线

最后更新：2026-06-25

## 产品愿景

`Word Adventures` 是一款用于记忆词汇的像素风剑与魔法 RPG。

核心产品价值是词汇学习。RPG 探索、战斗、装备、任务、奖励和故事是游戏化外壳，用来让反复回忆和复习变成冒险成长。

## 目标用户

- 第一目标用户：中国大陆小学和初中学生。
- UI 语言：默认使用简体中文。
- 英语只在它本身是学习对象时出现，例如词汇、拼写、选项，或刻意双语的物品名。
- 难度从小学 / 初中基础词开始。

## 当前引擎方向

- 引擎：Godot 4。
- 实现风格：2D 俯视角像素 RPG。
- 开发方式：视觉优先、测试支撑、小步迭代。
- Godot 项目规则见 `docs/production/godot-development-rules.md`。

## 当前产品验证方向

当前阶段回到“横屏 + 村庄”的上一版设定，优先在 PC Web 端验证核心可玩性。

当前主线：

- 画布和首要体验：横屏 `1280 x 720`。
- 首个可玩区域：第一章“书灯村”。
- 平台优先级：先 PC Web，后续再评估移动端或竖屏城堡。
- 操作方式：鼠标点击移动、鼠标点击交互按钮、键盘移动和 `空格/E` 交互并存。
- 验证目标：村庄探索、NPC 词汇挑战、装备/背包奖励、实时战斗、结算、章节转换是否形成顺畅闭环。

“单词城堡 / 竖屏 / 50 层”保留为长期方向参考，不作为当前开发主线。

## 视觉方向

本项目视觉优先。

规则：

- 玩家可见功能应尽量使用真实像素素材。
- 避免长期存在占位矩形或默认引擎风格 UI。
- UI 应该像打磨过的像素 RPG。
- 涉及布局或美术变化时，应该做截图或人工视觉 review。

当前视觉源头：

- `design/art/art-bible.md`
- `design/assets/asset-manifest.md`
- `design/ux/hud-and-prompts.md`

## 素材政策

允许的素材来源：

1. 本地已有的 Tiny Swords 授权素材。
2. CC0 / MIT / CC-BY 等许可安全的开源素材，例如 Kenney、LPC、Dungeon Crawl、OpenGameArt 素材包。
3. 当已批准素材库无法覆盖需求时，再考虑生成素材。

规则：

- 不提交 `external_assets` 中的付费原始素材包。
- 只把运行时真正需要的精选素材导入 Godot 项目。
- 每个素材来源都要记录到 `docs/asset-credits.md`。
- 使用规则记录到 `design/assets/asset-manifest.md` 和 `docs/visual-asset-usage.md`。

## 故事与第一章

长期故事框架：

- 世界由不同语言中的“真名”维持。
- 《万语之书》记录这些真名。
- 英语是第一套语言卷，但不是唯一语言卷。
- 长期目标支持 1500-3000 个词，并可扩展到日语、德语或自定义词库。

故事设计见：

- `docs/superpowers/specs/2026-06-21-story-script-design.md`

第一章是 20-30 分钟的“书灯村”。

核心前提：

- 《万语之书》的英语卷失控。
- 基础英语词变成散落书页、破碎句子和迷失词灵。
- 玩家通过村庄支线修复三页书页。
- 修复三页后，森林入口解锁 Word Imp 遭遇。

章节结构：

1. 图书管理员支线：释义识别。
2. 铁匠支线：拼写回忆。
3. 花园 / 水井支线：语境补全。
4. 森林入口 / Word Imp：第一次战斗高潮。

第一章玩法和剧本基线：

- `design/gameplay/beginner-village-loop.md`
- `docs/story/chapter-01-adventure-gate-village-script.md`
- `docs/story/chapter-01-02-map-design.md`

## 学习设计

词汇学习应通过以下方式发生：

- NPC 对话和短剧情语境。
- 释义识别题。
- 拼写输入或拼写组装。
- 句子语境补全。
- 将学会的词转化为可见的 RPG 成长奖励。

当前设计方向：

- 学习不需要打断每一次战斗动作。
- 学习应解锁装备、数值、任务进度或剧情能力。
- 战斗可以展示学习成果，但不强制每次攻击都回答单词题。

## 战斗方向

当前战斗方向是实时数值驱动 RPG 战斗。

早期回合制 / 单词卡战斗方案作为原型有价值，但当前方向是：

- 玩家和敌人通过动画和数值战斗。
- 攻速控制攻击频率。
- 攻击力、防御、生命、魔法、暴击和装备影响结果。
- 伤害以飘字显示。
- 敌人 / Boss 状态在战斗中靠近敌人显示。
- 战斗结算显示胜利 / 失败和奖励。

设计源头：

- `docs/superpowers/specs/2026-06-16-real-time-stat-combat-design.md`

《魔兽世界》主要作为以下方面参考：

- 数值成长。
- 装备品质语义。
- 角色 / 背包信息架构。
- 战斗可读性和奖励循环。

不要求复制：

- 精确公式。
- UI 美术。
- 完整职业、天赋或团队副本系统。

## 角色、装备与背包

第一版范围：

- 一个起始职业：战士。
- 一个统一快捷键同时打开角色面板和背包。
- 角色面板和背包窗口可通过标题栏拖动。
- 固定 20 格背包。
- 右键快速装备 / 使用。
- Tooltip 驱动物品理解。
- 新手村开放到 Rare / 精良品质语义。

设计源头：

- `docs/superpowers/specs/2026-06-19-equipment-inventory-design.md`

## 交互方向

在合理范围内，鼠标和键盘应该等价。

规则：

- 地图支持点击移动。
- 可交互 NPC / 怪物 / Boss 附近显示上下文提示。
- 提示左侧显示快捷键，例如 `空格/E`。
- 提示右侧显示清晰的行动 CTA，例如 `交谈` 或 `挑战`。
- 点击提示应触发和键盘交互相同的底层方法。

## 质量门槛

如果存在以下问题，变更不应进入玩家 review：

- 文本遮挡控件。
- UI 像默认 Godot 控件。
- 重要按钮缺少视觉对比。
- 精灵使用了半截或不匹配的素材。
- 角色因为视觉和碰撞不一致而卡住。
- 已反馈 bug 在可行时没有补回归测试。
