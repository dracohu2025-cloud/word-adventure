# Asset Manifest

This document tracks approved asset sources and project-specific usage rules.

Inspired by the asset workflow in [Claude-Code-Game-Studios](https://github.com/Donchitos/Claude-Code-Game-Studios), adapted for this project.

## License Policy

Allowed licenses:

- MIT
- CC0
- CC-BY
- Purchased commercial-use packs owned by the project owner

Avoid:

- Non-commercial licenses.
- Attribution terms that are unclear.
- Assets that cannot be redistributed in the target commercial build.

## Import Rules

- Do not upload paid source packs from `external_assets`.
- Only import the exact runtime assets needed into `assets/licensed` or another approved runtime folder.
- Add `.import` files generated by Godot only for assets included in the project.
- Keep source attribution in `docs/asset-credits.md`.
- Keep usage constraints in this file or `docs/visual-asset-usage.md`.

## Tiny Swords Compatible Generated Paths

Current selected runtime paths:

- `assets/generated/tiny_swords_compatible/props/`

Rules:

- These assets are project-generated companions, not official Tiny Swords files.
- Use this path when Tiny Swords does not include the required prop and fallback assets visibly clash.
- Keep each generated asset documented in `docs/asset-credits.md`.
- Validate generated assets in an in-game screenshot before accepting them.

## Fallback Asset Rules

Fallback assets are allowed when:

- Tiny Swords lacks the required object.
- A CC0 / MIT / CC-BY asset is visually compatible.
- The object is not a focal point or the mismatch is explicitly reviewed.

Fallback assets are not allowed when they create:

- Mixed outline thickness inside one focal object.
- Perspective mismatch.
- Partial-object artifacts.
- UI that feels from another game.

## Object Integrity Rules

Every visible asset must be used as a complete visual object.

Examples:

- A tree made of multiple sprites must include all required parts.
- A house must not have props accidentally covering its facade.
- A partial sprite from a sheet must not be placed as a standalone object.

If an object requires composition, document the full composition.

## Review Checklist For New Assets

- Source and license are known.
- Asset family matches the current scene.
- Pixel scale matches current camera and UI.
- Object is complete, not a cropped fragment.
- Collision or walkability matches the visual shape.
- The asset appears in a test scene or screenshot before being accepted.
