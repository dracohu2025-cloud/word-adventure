# Project Memory

最后更新：2026-06-25

这里记录会反复影响制作质量的项目级经验。长线程压缩或切换任务后，先读这里。

## 视觉与文案

- 用户是视觉依赖型，默认视觉优先。
- 玩家可见容器、边框、按钮、名牌必须素材化，不能临时代码绘制。
- 中文要自然、像游戏文案，不要像机器直译。
- 不要使用“新手入口村”这类占位式名称；地点、章节、道具都要有世界观内名字。
- 对话中的系统反馈应作为下一轮对话出现，不要在对话框里嵌入额外系统提示。

## 手机竖屏

- 单词城堡方向以竖屏 720x1280 为基准。
- 当前可视区域是大地图局部裁切，不是整张地图缩小预览。
- 手机上不出现“空格/E”等键盘提示。
- 顶部 HUD、小地图和底部物品栏浮在地图上，其余空间尽量给地图。

## 当前主线：横屏村庄

- 2026-06-25 起，当前主线回到“横屏 + 书灯村 + PC Web 可玩性验证”。
- 单词城堡竖屏方案保留为长期参考，不作为当前迭代默认方向。
- 横屏村庄阶段允许显示 `空格/E`，但鼠标点击交互也必须可用。
- 当前验收重点是村庄可玩闭环：探索、NPC、词汇挑战、奖励、装备背包、战斗、结算、章节转换。

## Godot 与验证

- 视觉改动必须截图自检。
- 文字边界问题要用代码断言和截图双重验证。
- 若截图与代码判断冲突，以截图为准。
- 新素材导入后要运行 Godot import。
